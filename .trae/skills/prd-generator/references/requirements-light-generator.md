# Light 需求文档生成 — 附录（对照表与长模板）

**主入口（独立技能）**：优先使用 **`prd-generator-light`**（`/.cursor/skills/prd-generator-light/SKILL.md`），可在对话里 **`@prd-generator-light`** 单独附加。

本文档保留：**与完整 PRD 的对照表**、**长版可复制指令**，供需要粘贴大块说明时使用。

用于：**快速产出「只做功能与交互」的需求说明**，不写背景故事、商业目标、用户画像长文、里程碑排期。产物落在 **`docs/prd/prd-<slug>-light.md`**（或与用户约定文件名）。

---

## 给用户：直接粘贴的开场指令（可按需改括号内容）

```text
@prd-generator-light
主题：<一句话功能名，例如 WebContainer 工作区导出导入>

请生成 PRD（Light）：
- 不要写：背景与问题、目标与成功指标、用户与场景、里程碑与依赖、长篇非功能。
- 必须写：极简 In/Out 范围；按界面模块拆分「功能与交互」（表格写清入口、步骤、状态）；每条下列验收 checklist；仅保留与交互直接相关的数据展示约定（列表列、排序）；文末「待定」表 +「修订记录」。
- 路径：docs/prd/prd-<kebab-slug>-light.md；若已有同名 Light 文件则增量修订并更新修订记录。
- 若仓库已有技术 spec（docs/specs/），属性表里链接过去，正文不重复 schema/API。
- 若提到具体页面按钮位置，在文档里给出代码路径锚点（例如 ** 组件文件相对路径 **）。
```

---

## 给 Agent：执行约束（与完整 PRD 的差异）

| 完整 PRD | Light PRD |
|----------|-----------|
| L0～L4 可多轮澄清 | 仍可做 **最多一轮** 澄清：仅缺交互关键点时再问（≤5 问） |
| 十章节模板 | **禁止**独立章节：背景、目标、用户场景、里程碑 |
| 非功能可展开 | 仅保留 **与交互绑定** 的条目（如 loading、禁用态、错误 Toast） |
| `prd-<slug>.md` | **`prd-<slug>-light.md`**（或与完整版互链） |

**正文推荐结构**

1. 标题 + 属性表（状态、完整版 PRD 链接可选、技术 spec 链接）
2. **范围（极简）**：包含 / 不包含，各若干条 bullet
3. **功能与交互**：按「入口 → Dialog/页 → 列表 → 行操作」分段；每段用表格（项目 | 说明）+ **验收** checklist
4. **数据与展示约定**：仅列用户可见字段与排序，schema 链 spec
5. **待定**：拍板前不写死实现
6. **修订记录**

**自检（Light）**

- [ ] 每条 Must 级交互都有可勾选验收项？
- [ ] In/Out 是否避免 scope creep？
- [ ] 待定项是否集中、可被评审一次性关掉？

---

## 与完整版 PRD 的并存策略

- 同一主题可先 **Light** 迭代交互，再按需补 **`prd-<slug>.md` 完整版**（背景、指标、场景）。
- Light 文首属性表建议链到完整版；完整版修订后可不在 Light 重复粘贴大段文字。
