# Requirements / 需求文档: quick-notes phase 1

## Restated Understanding / 需求复述

- 我理解当前需求是：在已初始化的 `apps/quick-notes` 桌面应用中，定义第一阶段业务功能的可评审需求边界。
- 当前核心目标是：让用户通过 Header 上的“任务 / 速记”两个 Tab 独立管理任务和速记；任务支持进行中与已完成，速记采用左侧列表 + 右侧编辑区。
- 当前边界是：只处理本机单窗口、单用户、应用数据目录 JSON 持久化下的基础增删改查和错误反馈。
- 暂不处理：账号体系、云同步、多设备同步、复杂标签、高级筛选、提醒通知、系统托盘、窗口快捷键、数据导入导出。

## Source Baseline / 来源基线

- 原始需求：`docs/prd/prd-quick-notes-light.md`
- 技术契约：`docs/specs/quick-notes-tauri-json.md`
- 应用目录：`apps/quick-notes`
- 当前项目事实：Tauri v2 + Svelte 5 + Vite + TypeScript + Tailwind CSS 4 框架已初始化；当前 `App.svelte` 仅为脚手架验证页，尚未实现 TODO、速记、本地 JSON 读写命令、前端业务 store 与业务页面交互。

## Scope / 范围

- In: 满窗体桌面工作区；Header 上的“任务 / 速记”两个 Tab；当前 Tab 内搜索；TODO 进行中队列；TODO 已完成归档；速记列表与编辑区；本地 JSON 读取、保存、读取失败重试、保存失败提示；空状态引导。
- Out: 登录/账号；云端或跨设备同步；复杂标签；高级筛选；跨 Tab 全局搜索；提醒通知；系统托盘；全局/窗口快捷键；数据导入导出；多窗口冲突处理；历史版本恢复。

## User Interaction / 用户交互

### 交互设计原则

- 物理使用场景：用户在桌面上处理工作或写代码时，被短任务、想法或临时备注打断，需要快速记录并立刻回到当前工作。
- 交互模型：任务和速记独立管理，用户通过 Header 上的两个 Tab 切换当前工作区。
- 页面结构：Header 只承载标题、Tab 与当前 Tab 搜索；主内容铺满窗体，避免居中小卡片式布局。
- 组件倾向：使用稳定的产品 UI 模式，例如 Tabs、Button、Collapsible、Checkbox、Dropdown、Textarea 或 inline action；不以弹窗作为首选交互。
- 反馈原则：正常操作尽量安静，新增、完成、恢复、删除通过列表变化反馈；只有读取失败、保存失败等会影响数据信任的情况需要明确提示。

### 应用壳

- 触发入口：用户启动 `quick-notes` 桌面应用。
- 用户操作路径：打开应用后进入满窗体桌面工作区；Header 上显示“任务 / 速记”两个 Tab，默认进入任务 Tab。
- 系统反馈：启动时读取本地数据；读取成功后展示当前 Tab 的内容与搜索入口；读取失败时展示可读错误和重试入口；当前 Tab 数据为空时展示对应空状态引导。
- 状态变化：加载中 -> 加载成功 / 加载失败；加载失败可通过重试再次触发读取。
- 异常/边界交互：读取失败时不白屏，不清空用户已有本地文件，不进入不可恢复状态。
- 不应发生的交互：不要求用户登录；不展示同步、导入导出、提醒或跨 Tab 全局搜索入口。

### Header Tab 与搜索

- 触发入口：用户点击 Header 上的“任务”或“速记”Tab，或在 Header 搜索框输入关键字。
- 用户操作路径：切换 Tab 后主内容区域只展示当前 Tab 的内容；搜索框始终针对当前 Tab 生效。
- 系统反馈：任务 Tab 下搜索过滤任务内容；速记 Tab 下搜索过滤速记正文内容和左侧摘要；搜索为空时恢复当前 Tab 全量列表。
- 状态变化：切换 Tab 不改变数据；搜索只改变当前视图过滤结果，不写入本地数据。
- 异常/边界交互：搜索无结果时展示轻量空结果文案；不把无结果误判为数据为空。
- 不应发生的交互：搜索跨 Tab 混合展示任务和速记；切换 Tab 后保留不适用于当前 Tab 的搜索结果。

### 任务 Tab

- 触发入口：用户进入任务 Tab，或在任务 Tab 中新增、完成、恢复、编辑、删除任务。
- 用户操作路径：新增任务；查看进行中任务；完成任务；查看已完成归档；恢复已完成任务；删除任务；按需编辑任务内容。
- 系统反馈：任务 Tab 内容铺满主区域；进行中任务作为默认主列表展示，按更新时间倒序排列并显示当前展示序号；任务行提供完成、编辑、删除等 inline action，右侧“完成”按钮与左侧勾选框完成效果一致；完成后任务从进行中移入已完成归档；已完成归档标题显示数量且默认折叠；展开后按完成时间倒序展示。
- 状态变化：新增任务为 `active`；完成任务变为 `done` 并记录 `completedAt`；恢复任务变回 `active` 并清空 `completedAt`；任一有效变更更新 `updatedAt` 并触发保存。
- 异常/边界交互：保存失败时保留前端内存中的当前状态并提示错误；用户可继续编辑，下一次有效操作再次触发保存。
- 不应发生的交互：空白任务进入列表；完成任务仍出现在进行中队列；恢复后仍保留完成时间；任务 Tab 出现速记编辑区；删除 TODO 影响速记数据。

### 速记 Tab

- 触发入口：用户进入速记 Tab，或在速记 Tab 中新增、选择、编辑、删除速记。
- 用户操作路径：新增速记；在左侧列表选择速记；在右侧编辑区通过 Milkdown Crepe 编辑器查看和编辑当前速记 Markdown 正文；删除当前速记。
- 系统反馈：速记 Tab 使用 sidebar + content 模式；左侧列表按更新时间倒序展示速记标题与摘要；右侧展示当前选中速记的可编辑正文；新增或编辑后左侧列表按更新时间更新排序；删除后从列表移除并选择下一条可用速记。
- 状态变化：新增速记记录 `createdAt` 与 `updatedAt`；编辑速记更新 `content` 与 `updatedAt`；删除速记只影响 notes 集合；任一有效变更触发保存。
- 异常/边界交互：保存失败时保留当前输入或内存状态，并给出可读错误。
- 不应发生的交互：用户手动输入速记标题；空白速记进入列表；删除速记影响 TODO 数据；速记 Tab 出现进行中/已完成状态。

### 速记编辑器 / Milkdown Crepe

- 编辑器来源：按官方文档 `https://milkdown.dev/docs/guide/using-crepe` 接入 `@milkdown/crepe`。
- 内容格式：速记 `content` 仍保存为 Markdown 字符串；Crepe 只负责编辑体验，不改变 `QuickNote` 数据结构。
- 初始化：选择或新增速记时，编辑器应以当前正文作为 `defaultValue`；切换速记或退出新增态时应避免旧实例残留。
- 同步策略：编辑器变更通过 Crepe 的 Markdown 更新事件回传给速记编辑区草稿；点击保存时沿用现有 `onCreateNote` / `onUpdateNote` 流程。
- 样式策略：导入 Crepe 官方 common 样式和一个内置主题；再用当前 Tailwind token 做最小容器适配，保持右侧编辑区铺满高度。
- 边界：本轮不做图片上传、附件管理、云端资源代理、自定义 Crepe 插件、多人协作或 Markdown/所见即所得双模式切换。

### 速记标题与摘要

- 标题来源：速记标题由正文自动生成，优先取正文第一行 trim 后的内容。
- 摘要来源：左侧列表正文摘要由正文自动生成，优先摘取第一行；当第一行过长时做单行截断。
- 空内容：正文为空或 trim 后为空时不可保存；未命名速记不进入列表。
- 展示策略：标题和摘要仅用于左侧列表识别，不作为独立可编辑字段。

### 状态与错误反馈

- 读取失败：展示错误说明与“重试读取”入口；不展示空数据假象。
- 保存失败：在页面内展示轻量错误提示和“重试保存”入口；用户当前输入和内存状态必须保留。
- 正常保存：默认静默保存；不要求持续展示“保存中”或“已保存”状态。
- 删除：首期直接删除，不弹出二次确认；删除操作必须只影响当前项目类型。

## Data Requirements / 数据需求

- 数据文件位于 Tauri 应用数据目录下的 `quick-notes.json`。
- 文件不存在时视为空数据：`tasks: []`、`notes: []`。
- TODO 字段：`id`、`content`、`status`、`createdAt`、`updatedAt`、`completedAt`。
- 速记字段：`id`、`content`、`createdAt`、`updatedAt`。
- `content` 保存前 trim，trim 后为空则不提交。
- 时间字段使用 ISO 字符串，由前端生成后交给 Tauri 保存。
- 首期不做复杂冲突处理；同一窗口内以最新一次有效操作覆盖保存。

## Constraints / 约束

- 业务约束：第一阶段以快速记录和轻量任务流转为主，不引入复杂任务管理能力。
- 技术约束：业务实现必须遵守 `docs/specs/quick-notes-tauri-json.md` 中的数据模型、Tauri 命令和写入策略；`core` 可复用逻辑优先使用类封装。
- UI 约束：沿用仓库 Tailwind 4、Bits UI / shadcn-svelte 风格；当前需求未提供 Figma 或视觉稿，因此不创建 `visual-design.md`。
- 数据约束：本地 JSON 为单一真相源；首期不做 schema migration；同一窗口内最新操作覆盖保存。
- 兼容性约束：优先保证 Windows 桌面构建链路；若 Tauri 原生构建依赖缺失，至少完成 Svelte/Vite 层验证并记录原因。

## Open Questions / 开放问题

- [x] Q1 删除 TODO 或速记时是否需要二次确认，还是首期直接删除？
  - 处理：首期直接删除，不弹出二次确认。
- [x] Q2 保存失败提示是否需要提供显式“重试保存”按钮，还是通过下一次编辑/操作再次触发保存即可？
  - 处理：保存失败时提供显式“重试保存”入口；正常保存默认静默，不展示持续状态。
- [x] Q3 TODO 内容是否需要支持编辑？原始 PRD 未明确，仅要求新增、完成、恢复、删除。
  - 处理：支持任务内容内联编辑，保持与速记编辑一致。
- [x] Q4 速记编辑采用列表内直接编辑，还是点击后进入独立编辑态？
  - 处理：采用 sidebar + content 模式；左侧选择速记，右侧编辑当前速记正文。
- [x] Q5 已完成区默认折叠是每次启动都折叠，还是记住用户上一次展开状态？
  - 处理：首期每次启动默认折叠，不记忆展开状态。
- [x] Q6 错误提示展示为页面内横幅、toast，还是字段/区域内提示？
  - 处理：使用页面内轻量错误提示；保存失败靠近当前操作区域或主内容区域，读取失败占据主内容区域。
