# Requirements / 需求文档: quick-notes pin and delete actions

## Goal / 目标

为 `apps/quick-notes` 新增更明确的列表级操作：

- 便签删除改为在左侧便签列表行右侧以 hover icon 呈现，用户点击哪一行就删除哪一行。
- 删除当前选中便签后只清空选中 id，不自动选择下一条便签。
- 为进行中任务和便签增加置顶能力。
- 置顶不改变当前主列表展示；置顶条目仍留在原列表，同时在列表上方的置顶折叠区重复展示。

## Scope / 范围

### In

- 进行中任务置顶/取消置顶。
- 便签置顶/取消置顶。
- 任务 Tab 的主列表上方新增“置顶”折叠区，默认折叠。
- 便签左侧列表上方新增“置顶”折叠区，默认折叠。
- 便签行 hover 展示删除 icon；删除 icon 点击时不触发行选择。
- 删除当前选中便签后，右侧编辑区进入未选择状态。
- 本地 JSON 数据新增可选置顶字段，并兼容旧数据。

### Out

- 已完成任务不支持置顶。
- 置顶不从主列表移除条目，不改变主列表排序。
- 不做拖拽排序、自定义置顶顺序或多级置顶。
- 不改变 Tauri `load_store` / `save_store` 命令形态。
- 不引入确认弹窗。

## Functional Requirements / 功能需求

### 便签删除

- 每条便签列表行右侧提供删除 icon。
- 删除 icon 默认弱化或隐藏，仅在 hover/focus 时明显展示。
- 点击删除 icon 只删除该行便签，不选择该便签。
- 删除非当前选中便签时，当前选中和右侧编辑器不受影响。
- 删除当前选中便签时，`selectedNoteId` 置为 `null`，右侧显示“选择或新增一条速记”空状态。

### 任务置顶

- 仅进行中任务展示置顶/取消置顶入口。
- 点击置顶后，为任务记录置顶时间。
- 点击取消置顶后，清除任务置顶状态。
- 置顶不会改变进行中任务主列表排序和展示。
- 任务主列表上方展示“置顶”折叠区，默认折叠；展开后按置顶时间倒序展示置顶任务。

### 便签置顶

- 每条便签列表行提供置顶/取消置顶入口。
- 点击置顶后，为便签记录置顶时间。
- 点击取消置顶后，清除便签置顶状态。
- 置顶不会改变便签主列表排序和展示。
- 便签列表上方展示“置顶”折叠区，默认折叠；展开后按置顶时间倒序展示置顶便签。

## Data Requirements / 数据需求

- `QuickTask` 新增可选字段：`pinnedAt?: string | null`。
- `QuickNote` 新增可选字段：`pinnedAt?: string | null`。
- 置顶时写入当前 ISO 时间字符串。
- 取消置顶时写入 `null` 或移除等价空值；保存前统一为稳定结构。
- 读取旧数据时，缺失 `pinnedAt` 的任务/便签视为未置顶。

## Acceptance Criteria / 验收标准

- [ ] 删除当前选中便签后，右侧编辑器不再展示被删除便签内容，也不会自动切换到下一条。
- [ ] 删除非当前选中便签后，当前编辑器保持原选中便签。
- [ ] 进行中任务可置顶/取消置顶；已完成任务不展示置顶入口。
- [ ] 便签可置顶/取消置顶。
- [ ] 置顶条目在主列表中仍保留，主列表顺序不因置顶改变。
- [ ] 置顶折叠区默认折叠，展开后显示置顶条目。
- [ ] 置顶折叠区按 `pinnedAt` 倒序展示。
- [ ] 关闭并重新打开应用后，置顶状态能从本地 JSON 恢复。
- [ ] `pnpm --filter quick-notes check` 通过。
- [ ] `pnpm --filter quick-notes build` 通过或仅保留既有非阻塞 warning。
