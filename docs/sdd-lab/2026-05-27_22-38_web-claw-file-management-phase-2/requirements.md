# Requirements / 需求文档: Web Claw File Management Phase 2

## Restated Understanding / 需求复述

- 我理解当前需求是：在文件管理一期目录树、上下文菜单、文本读写能力之上，推进二期文件操作能力。
- 当前核心目标是：让目录树容器空白区域也具备上下文菜单；支持上传、下载、复制与粘贴文件/目录，形成更完整的 IDE 风格文件管理闭环。
- 当前边界是：先定义用户交互、范围、验收标准和约束；不在本阶段设计具体 TypeScript 接口、BrowserPod 命令细节或 UI 组件实现。
- 暂不处理：多工作区、多盘、拖拽批量目录上传、剪切移动、跨浏览器标签页共享剪贴板、系统剪贴板文件互通、版本历史、压缩包预览、冲突自动合并。

## Related Iterations / 关联迭代

- `2026-05-26_12-00_web-claw-file-management`：一期文件管理；二期复用其 Files Tab、目录树、当前工作区路径、runtime session、文件服务边界和按需同步心智。

## Scope / 范围

### In

- 扩展目录树上下文菜单触发区域：
  - 在目录树 item 上打开菜单时，操作目标为该文件或目录。
  - 在目录树容器空白区域打开菜单时，操作目标为当前工作区路径。
- 支持上传文件到当前操作目标：
  - 在工作区或目录目标上上传时，文件进入对应目录。
  - 在文件目标上上传时，默认以该文件父目录作为上传目标。
  - 上传完成后刷新受影响目录。
- 支持下载文件：
  - 文件目标支持下载该文件。
  - 是否支持目录下载由技术方案阶段确认；需求阶段不默认承诺。
- 支持复制粘贴文件与目录：
  - 文件目标支持复制。
  - 目录目标支持复制。
  - 目录或工作区目标支持粘贴到该目录。
  - 文件粘贴同名冲突时，不弹二次确认、不覆盖，自动探测并使用第一个不存在的副本目标名：先尝试 `{name}_copy`，若仍存在则继续尝试 `{name}_copy_2`、`{name}_copy_3` 等。
  - 目录粘贴同名冲突时，不弹二次确认、不覆盖、不合并，自动探测并使用第一个不存在的副本目标名：先尝试 `{name}_copy`，若仍存在则继续尝试 `{name}_copy_2`、`{name}_copy_3` 等。
  - 粘贴完成后刷新目标目录。
- 文件列表读取改用已注入的 `uls` 指令，避免继续扩大对系统 `ls` 输出的依赖。
- Files 工具栏提供显示隐藏文件的 checkbox：
  - 默认不显示 dotfile。
  - 勾选后刷新目录列表并显示隐藏文件。
  - 不显示 `.` 和 `..` 这类导航伪目录。
- 定义冲突、失败、容器不可用、权限或路径异常时的用户可见反馈。

### Out

- 不实现剪切 / 移动语义；重命名仍属于一期能力。
- 不承诺浏览器系统剪贴板中的文件与应用内文件剪贴板互通。
- 不承诺大文件分片上传、断点续传、批量上传进度队列。
- 不引入后台实时监听；仍沿用用户交互触发按需读取最新信息。

## User Interaction / 用户交互

### 触发入口

- 用户在 Files Tab 左侧目录树区域点击右键或使用键盘上下文菜单。
- 用户在文件或目录 item 上触发上下文菜单。
- 用户在目录树容器空白区域触发上下文菜单。

### 用户操作路径

1. 用户在目录树 item 上打开上下文菜单，菜单操作以该 item 为目标。
2. 用户在目录树容器空白区域打开上下文菜单，菜单操作以当前工作区路径为目标。
3. 用户选择上传文件，系统打开浏览器文件选择器；选择文件后上传到目标目录，并刷新该目录。
4. 用户选择下载文件，浏览器触发文件下载；失败时展示错误反馈。
5. 用户在文件或目录上选择复制，系统记录应用内文件剪贴板状态，并在 UI 上可感知当前已复制对象。
6. 用户在目录或工作区目标上选择粘贴，系统把已复制文件或目录复制到目标目录；完成后刷新目标目录。
7. 若粘贴文件时目标目录已有同名文件，系统自动生成第一个不存在的副本目标名，不弹二次确认、不覆盖原文件。
8. 若粘贴目录时目标目录已有同名目录，系统自动生成第一个不存在的副本目标名，不弹二次确认、不覆盖、不合并原目录。

### 系统反馈

- 上传、下载、复制、粘贴执行中应有可见状态，避免重复提交同一操作。
- 操作成功后应展示明确反馈，并刷新受影响目录。
- 操作失败时应展示可读错误，不静默失败。
- 容器非 `running` 时禁用文件操作，并复用一期阻塞态说明。
- 用户切换“显示隐藏文件”后，已展开目录应按新选项重新读取，保持展开状态不丢失。

### 不应发生的交互

- 不得只有目录树 item 能打开上下文菜单；空白区域必须能打开工作区菜单。
- 不得在空白区域上下文菜单中误用最近选中文件作为操作目标。
- 不得上传到文件路径本身；文件目标下的上传应落到父目录。
- 不得在未确认冲突策略时静默覆盖已有文件。
- 不得把复制状态写入浏览器系统剪贴板，除非后续需求明确要求。

## Constraints / 约束

- 业务约束：二期仍是单容器、单当前工作区路径语义；空白区域操作目标为当前工作区，不是页面根容器或终端 cwd。
- 技术约束：继续遵守 `apps/web-claw` / `packages/os-core` / `packages/browserpod` 分层；app 不暴露 BrowserPod SDK 类型。
- 交互约束：右键菜单目标必须可解释、可预测；不得因当前选中态导致空白区域操作目标漂移。
- 安全约束：不提交用户上传文件内容到仓库；上传下载失败信息不得泄漏敏感 token。
- 兼容约束：第一轮验证以桌面 Chromium 系浏览器为基线。

## Requirement Decisions / 需求决策

- 2026-05-27 22:38:
  - 决策：创建独立二期 SDD 迭代 `web-claw-file-management-phase-2`，需求状态按用户确认写入。
  - 原因：一期仍处于执行与真实环境验证阶段；右键菜单作用域、上传下载、复制粘贴属于新的文件操作范围，独立迭代更便于评审和执行门禁。
- 2026-05-27 22:54:
  - 决策：目录复制粘贴纳入二期；目录粘贴目标同名冲突时自动使用 `{name}_copy` 作为新目标目录名。
  - 原因：目录复制实现成本可控；用副本名可避免覆盖或合并目录带来的数据风险。
- 2026-05-27 23:16:
  - 决策：粘贴操作不需要用户二次确认；文件和目录粘贴同名冲突均使用 `{name}_copy` 作为新目标名。
  - 原因：用户已经通过 Paste 表达明确意图；自动副本名避免确认打断，同时避免静默覆盖。
- 2026-05-27 23:26:
  - 决策：粘贴目标存在时必须由系统持续探测并选择第一个不存在的副本目标名；不能只尝试一次 `{name}_copy`。
  - 原因：`{name}_copy` 也可能已存在；复制应尽量自动成功，避免要求用户多次处理冲突。
- 2026-05-29 03:06:
  - 决策：文件管理目录列举改用注入的 `uls`，并在 Files 工具栏增加“显示隐藏文件”checkbox；默认隐藏 dotfile，勾选后显示隐藏文件但排除 `.` / `..`。
  - 原因：`uls` 用于修复容器内文件名显示问题；隐藏文件属于用户可控的浏览偏好，不应默认增加目录噪音。
