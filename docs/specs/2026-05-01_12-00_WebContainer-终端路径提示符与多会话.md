# WebContainer 演示 — 终端：路径提示符 + 多会话

## 目标

1. **类 Linux 提示符**：在 xterm 输入行展示「当前工作目录 + `$`」；目录与会话内 `spawn({ cwd })` 使用的相对路径一致（相对 WebContainer `workdir`）。
2. **多终端**：支持多个终端标签；每个标签独立的 xterm、日志环、进程/stdin 引用与忙状态；共享同一 `WebContainer` 单例与已挂载 workspace。

## 边界 / 非目标

- 不引入真实长驻 PTY shell；仍为「每行 `sh -c`」模型。
- **仅当整行**为 `cd`（可选参数）时，在命令成功（exit 0）后更新会话 cwd；`cd foo && cmd` 等形式不改变会话 cwd（与 one-shot shell 语义一致）。
- 不创建多个 WebContainer 实例。

## Done Contract

- 首屏与每条命令结束后出现可继续输入的「路径 + `$`」提示符；键入可见，回车执行后输出与提示符顺序正确。
- 新建第二个终端标签后，两标签可交替输入；一标签 `busy` 时不应阻塞另一标签的输入与执行。
- `cd` 仅行成功后可改变该标签后续命令的 spawn cwd。
- 校验：`pnpm -C demos/webcontainer-openclaw build`（或等价 `npm run build`）通过。
