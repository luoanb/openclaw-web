# WebContainer 演示 — 多终端侧栏样式优化

## 关联与边界

- **关联**：[《终端路径提示符与多会话》](./2026-05-01_12-00_WebContainer-终端路径提示符与多会话.md)（多标签行为、单 WebContainer、每标签独立会话）。
- **本 spec 范围**：仅 **终端标签条 UI** 的布局与视觉（`TerminalPanel` 内会话列表）；**不**改 `TerminalPane` 内 spawn、提示符、`core/terminal` 契约。
- **非目标**：不改「运行 / 预览」顶层 `ShellTabs`；不新增依赖（如无则不用图标库，用可访问的字符或内联 SVG）。

## 目标（验收）

1. **布局**：多终端会话列表置于 **终端内容区容器右侧**，**纵向**排列；**xterm 区域在左侧**（或主区域），占满剩余宽度（`flex-1 min-w-0`），避免挤压终端。
2. **关闭**：默认不展示关闭控件；**鼠标悬停在该会话行上**时于 **行内右侧** 展示 **类似 × 的关闭图标**（与同一行的 hover 底同属一行容器，非并列分栏）；可 `aria-label`「关闭 {标签名}」；仅当 `tabs.length > 1` 时允许关闭。
3. **状态弱化**：**选中**与 **hover** 不使用强 ring、粗描边、大阴影；仅用 **轻微背景色差异** 区分默认 / hover / 选中（沿用或微调现有 CSS 变量，如 `--tab-rail-bg`、`--surface-*`，避免高对比「药丸」强态）。

## Research Findings（事实）

- 当前实现：`demos/webcontainer-openclaw/src/lib/features/terminal/components/TerminalPanel.svelte` 顶部 `flex-wrap` 横排 + `ring-2` 选中 + 常显「×」分栏按钮。
- 设计索引：`docs/design/BitsUI.md`（Tabs 参考在 `ShellTabs`，本改为侧栏列表，语义为「会话切换」，非顶层 Bits Tabs）。
