## 1. Requirements (Context)

### Goal

在 **`demos/webcontainer-openclaw`** 终端面板实现 PRD Light 所述 **「文件」** 入口：Dialog 内 **「导出」DropdownMenu（每项为导出动作）**、**已导出列表**（路径 / 时间 / 类型）、行 **hover 导入**；导入后 **mount** 并 **复用一键 PoC 节后链路**；**导入不论成败不删除** IndexedDB 记录。

### In Scope

- 工具栏按钮 **文件**、`TerminalPane` 内 Dialog 与列表交互。
- 调用 `WebContainer#export` / `mount` 与 `WorkspaceTreeIdbStore`（契约见 **§4.10**）。
- **导出交互**：**DropdownMenu**，触发器 **「导出」**；菜单 **每一项点击即执行**对应 `exportKind` 的导出（非 Select、非「选类型 + 独立导出按钮」）。
- MVP 菜单动作 **`json-tree`** ↔ `export(path, { format: 'json' })`。
- **导出根路径** MVP：固定 `'.'`（整工作区相对 `workdir`）；排除规则 **本期不做**（与 PRD「待定」一致，避免静默变更树大小策略）。
- **重启项目**：导入成功后顺序执行与 **`onPoc` 相同的 `npm install` + `npx openclaw --help`**（同一 `spawnCwd` 规则），**不再次** `WebContainer.boot()`。
- **导入二次确认**：MVP **不做**（与 PRD「待定」取默认；后续可加）。

### Out of Scope

- PRD Light 已列：云同步、多 Tab 并发、自动定时导出、JSON 二进制可逆等。
- **删除快照** UI：本期不要求（存储层 `deleteSnapshot` 可不暴露）。

### 1.1 Context Sources

- PRD Light（验收条目逐项可追溯）。
- 本文 **§4.10**（IndexedDB 与快照结构）。
- 代码锚点：[`TerminalPane.svelte`](../../demos/webcontainer-openclaw/src/lib/features/terminal/components/TerminalPane.svelte)（`onPoc`、工具栏）、[`boot.ts`](../../demos/webcontainer-openclaw/src/lib/core/webcontainer/boot.ts)（`bootWebContainer`、`ensureWorkspace`）。

## 2. Research Findings

| 事实 | 说明 |
|------|------|
| `onPoc` 流水线 | `bootWebContainer` → `wirePreview` → `ensureWorkspace`（初始 `tree`）→ `npm install` → `npx openclaw --help`；使用 `WebContainerShellRunner.runSpawn` 与 `spawnCwd`。 |
| **二次 mount 风险** | `ensureWorkspace` 使用模块级 `workspaceMounted`，**仅首次 mount**。导入快照后需再次 `wc.mount(...)`，当前实现会 **跳过**，与导入语义冲突。 |
| **决议（写入 Plan）** | Execute 阶段必须 **解除或绕过**该短路：例如增加 `forceRemountWorkspace(wc, tree)` / 导入前重置标志位 / 将「初始模板 mount」与「用户工作区 mount」分层（**具体补丁以 Plan checklist 为准**）。 |
| Dropdown 组件 | demo 内暂无 `Dropdown*.svelte`；须 **新增** `$lib/ui` 封装或直接使用 **`bits-ui`** Dropdown（样式对齐 `BitsUI.md` Figma）。 |

### 风险

| 风险 | 缓解 |
|------|------|
| 大仓库 `export` / IDB 写入阻塞 | 导出按钮 loading/disabled；Toast `QuotaExceededError`（`IdbQuotaError`）。 |
| `mount` 与运行中进程 | 导入前若存在前台 shell，行为与现有一致：文档声明「导入不自动中止」或 Execute 时加门禁（与 PRD协调，默认 **不中止**）。 |

## 4. Plan (Contract)

### 4.1 常量与命名空间

| 项 | 值（MVP） |
|----|-----------|
| `workspaceKey` | 单一常量，例如 `'wc-openclaw-demo'`（文件：`core/webcontainer/workspaceConstants.ts` 或并入现有配置模块）。 |
| `exportRootPath` | 固定 `'.'`。 |
| 导出菜单项（动作） | 与 `@webcontainer/api` 一致三项：`json-tree`、`binary-snapshot`、`zip-archive`（文案见 `EXPORT_KIND_LABELS`）；**每项点击即执行导出**。`zip-archive` 快照不支持导入回容器。 |

### 4.2 UI 结构（须符合 PRD 验收）

1. **工具栏**：`中止` → `一键 PoC` → **`文件`**（[`TerminalPane.svelte`](../../demos/webcontainer-openclaw/src/lib/features/terminal/components/TerminalPane.svelte)）。
2. **Dialog**：标题建议「工作区文件」；`Esc`/关闭即关；异步进行中可用 Toast 反馈。
3. **区块顺序**：**「导出」`DropdownMenu`**（触发器打开菜单；**菜单每项 = 一种导出动作**，点击即跑导出链路）→ **列表**（空态文案见 PRD）。
4. **列表列**：导出路径（`exportRootPath` 展示）、导出时间（格式化为本地可读字符串）、导出类型（由上表 label 映射 `exportKind`）。
5. **行 hover**：`json-tree` / `binary-snapshot` 右侧显现 **导入**；`zip-archive` **不提供导入**（右侧常驻 muted 说明）。**键盘可达性** Should（`focus-within` 同现有多终端侧栏关闭按钮模式）。
6. **就绪态**：`WebContainer` 未就绪或 `crossOriginIsolated === false` 时，禁用导出/导入并提示（「文件」按钮是否仍可打开 **允许**，仅列表空 + disabled）。

### 4.3 导出流程（顺序）

1. 用户点开 **「导出」** 触发器，在菜单中点击某一 **导出动作**（`json-tree` / `binary-snapshot` / `zip-archive`）。
2. 按 `exportKind` 调用 `wc.export(exportRootPath, { format })`：`json-tree` → `format: 'json'` 得 `FileSystemTree`；`binary-snapshot` → `format: 'binary'`；`zip-archive` → `format: 'zip'` —— 后二者得 `Uint8Array`，转 `ArrayBuffer` 入库。
3. 存储层校验：`json-tree` 执行 `assertValidFileSystemTree`；二进制类校验 `binaryPayload.byteLength > 0`。
4. `putSnapshot`（ discriminated input：`tree` 或 `binaryPayload`）→ `snapshotId`。
5. `listSnapshots(workspaceKey)` 刷新 UI。

**失败**：Toast；**不得**写入半条记录（事务或先内存组装再 put，与 **§4.10** 一致）。

### 4.4 导入流程（顺序）

1. `getSnapshot(snapshotId)`（**拒绝** `zip-archive`，UI 不提供入口）。
2. `wc.mount(payload, { mountPoint })` — `payload` 为 `FileSystemTree`（`json-tree`）或 `ArrayBuffer`（`binary-snapshot`）；**`mountPoint` 语义见 §4.10.4**；MVP `exportRootPath === '.'` 时等价当前 `mount(...)` 行为。
3. **禁止** `deleteSnapshot(snapshotId)`（成功/失败均禁止）。
4. **重启项目**：调用与 `onPoc` **共享**的 **`npm install` + `npx openclaw --help`** 序列（参数、`spawnCwd`、`busy`/Toast 行为一致）；**不**重复 `boot`。
5. Toast 成功或失败；失败时列表记录仍存在。

### 4.5 `ensureWorkspace` / 二次 mount（强制）

- 导入路径在 **执行 mount 之前** 必须保证 **不会对「已 mount」错误短路**：须修订 [`boot.ts`](../../demos/webcontainer-openclaw/src/lib/core/webcontainer/boot.ts)（或等价机制），使 **同一 `wc` 实例可再次 mount 导入树**。
- 初始模板 mount（首次进入 demo）与 **导入覆盖** 的语义须在代码注释中写清，避免回归。

### 4.6 依赖组件

- **导出：DropdownMenu（动作语义）**：在 `WorkspaceFilesDialog` 内联 **`bits-ui` `DropdownMenu`**：`Trigger` 文案 **「导出」**（不承载「当前选中类型」字段语义）；`Content` 内 **`Item`** 每项绑定一种 **`exportKind`**，**`onSelect` 即执行**导出流水线（与 §4.3 一致）。**不得**使用 Select /「类型下拉 + 单独导出按钮」表达同一交互。对照 **`BitsUI.md`** Dropdown 节点。
- **Dialog**：优先复用已有 `$lib/ui` Dialog/Modal；若无则按 Bits UI 补。

### 4.9 Validation（证据）

- 同一 `workspaceKey` 连续导出两次 → 列表两条，`savedAt` 降序。
- 导入失败（例如故意损坏 snapshot 仅在读库后校验场景）→ Toast + **IDB 记录仍在**。

### 4.10 IndexedDB 存储契约（唯一真相源）

本节约定 **工作区快照** 在浏览器 IndexedDB 中的模型与 API，与 `WebContainer#export` / `mount` 协作；**不依赖**任何已废止文档。

#### 4.10.1 与 `export(json)` 对齐的树类型

与 `@webcontainer/api` 的 `FileSystemTree` 一致；实现层须导出类型守卫与 `assertValidFileSystemTree`（读库后校验，防损坏数据）。**禁止**在 `tree` 节点内写入 SDK 未定义字段。

```typescript
export type FileSystemTree = Record<string, FileSystemTreeEntry>;
export type FileSystemTreeEntry = FileLeaf | DirectoryLeaf;
export interface FileLeaf {
  file: { contents: string };
}
export interface DirectoryLeaf {
  directory: FileSystemTree;
}
```

**已知限制**：非 UTF-8 / 二进制经 `format: 'json'` **不保证字节级可逆**（与 WebContainers 语义一致）。

#### 4.10.2 快照记录（Envelope）

```typescript
/** 与 `WebContainer#export` 的 `ExportOptions.format` 对齐：`json` | `binary` | `zip`。 */
export type WorkspaceExportKind = "json-tree" | "binary-snapshot" | "zip-archive";

export interface WorkspaceTreeSnapshotRecord {
  schemaVersion: 2;
  snapshotId: string;
  workspaceKey: string;
  exportRootPath: string;
  exportKind: WorkspaceExportKind;
  /** `exportKind === "json-tree"` 时必填；其余格式不写此字段。 */
  tree?: FileSystemTree;
  /** `binary-snapshot` / `zip-archive` 时必填（`Uint8Array` 在 IDB 中以 `ArrayBuffer` 存储）；`json-tree` 不写。 */
  binaryPayload?: ArrayBuffer;
  savedAt: number;
  contentRevision?: string;
}
```

**与官方 API 一致的限制**：`format: 'zip'` 的产物为 ZIP 字节流，**不得**作为 `mount()` 参数还原工作区（官方说明仅 **`json` 与 `binary`** 可作为 `mount` 输入）。UI 对 `zip-archive` 快照**不提供导入**，仅保留列表与导出能力。

#### 4.10.3 IndexedDB 物理结构（Version 2）

| 项 | 值 |
|----|-----|
| Database name | 默认 `'openclaw-wc-workspace-file-system'`（可构造参数覆盖） |
| Version | `2` |
| Object store | `snapshots` |
| keyPath | `snapshotId` |
| 索引 | `workspaceKey`、`savedAt`（列举与排序） |

同一 `workspaceKey` **多条**记录；`listSnapshots(workspaceKey)` 返回按 **`savedAt` 降序**。

#### 4.10.4 `mount` 语义（调用方）

- `exportRootPath` 用于审计与列表展示；恢复时：若导出根为 `'.'`，通常 `wc.mount(tree | binary)` 与现有一致；若为子路径（本 MVP 外），须 `mount(..., { mountPoint: <与 export 一致> })`。
- **ZIP**：官方 API 仅声明 **`json` / `binary`** 可作为 `mount` 输入；ZIP 字节流不得走导入链路。

#### 4.10.5 存储类 `WorkspaceTreeIdbStore`（概念 API）

| 方法 | 行为 |
|------|------|
| `open()` / `close()` | 打开 DB version 2，创建 store 与索引 |
| `putSnapshot(input)` | 填充 `schemaVersion`、`savedAt`；按 `exportKind` 校验 `tree` 或 `binaryPayload`；返回 `snapshotId` |
| `getSnapshot(snapshotId)` | 读出并校验 |
| `listSnapshots(workspaceKey)` | 降序 `savedAt` |
| `deleteSnapshot(snapshotId)` | **仅**显式删除入口；**导入流程禁止调用** |

**错误类型**：`InvalidSnapshotError`（树无效）、`IdbQuotaError`（配额）。

#### 4.10.6 与其它章节交叉引用

- §4.3 / §4.4 导出导入步骤须遵守本节；多 Tab 并发、端到端加密等 **本期不做**。
