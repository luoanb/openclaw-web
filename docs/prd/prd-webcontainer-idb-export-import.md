## 1. 背景与问题

- PoC 场景下，用户在浏览器内修改 WebContainer 工作区后，刷新或关闭页面即丢失；需要**低成本持久化**与**一键还原**。
- 工程侧已规划与 `export(json)` 同构的 IndexedDB 存储 API，产品侧需定义**入口、列表呈现与关键操作闭环**（导出 → 列表 → 导入并恢复运行）。

## 2. 目标与成功指标

| 类型 | 内容 |
|------|------|
| 北极星 | 用户无需离开 demo 即可完成「保存当前工作区快照 → 稍后从列表还原并继续 PoC」。 |
| 辅助指标 | 导出/导入操作可在 **3 步内**完成（打开对话框 → 导出或选择记录导入 → 确认结果）；失败时有可理解提示（配额、结构化克隆限制等由实现参照技术 spec）。 |

## 3. 范围

### 3.1 本期包含（In Scope）

- 在 **一键 PoC 右侧**新增 **「文件」** 按钮；点击打开 **Dialog**。
- Dialog 内：**已导出记录列表** + **导出** 操作入口。
- 列表 **列**：**导出路径**（导出时所选根路径或约定展示字段）、**导出时间**（用户表述为「导出事件」，本产品口径解释为**事件发生时间**，精确到对用户有意义的粒度，如本地日期时间字符串）。
- **行级轻交互**：鼠标 **hover** 某条记录时，在**行右侧**露出 **「导入」**（轻量按钮或文字按钮，避免默认占位过重）。
- **导入**：点击导入 → **将快照挂载回容器**，并 **重新启动项目**（与当前 PoC「可运行态」一致；精确语义见 §5.3 开放问题）。

### 3.2 本期不包含（Out of Scope）

- 跨浏览器 / 跨设备的账号云同步。
- 二进制字节级可逆导出（参见技术 spec：JSON 路径下非 UTF-8 **不保证可逆**）。
- 多 Tab 并发写入冲突治理（技术 spec MVP 不纳入）。
- **自动定时导出**（除非后续评审升格为 Must；当前仅保证 Dialog 内显式导出）。

## 4. 用户与场景

### 4.1 角色概览

| 角色 | 目标 |
|------|------|
| Demo / PoC 使用者 | 保存与还原工作区，减少重复搭建成本。 |

### 4.2 用户故事

| 角色 | 我想要 | 以便于 |
|------|--------|--------|
| PoC 使用者 | 点「文件」看到历史导出 | 选择合适快照还原 |
| PoC 使用者 | 一键导出当前工作区 | 写入 IndexedDB 备援 |
| PoC 使用者 | 在某条记录上 hover 并点导入 | 挂载并回到可演示状态 |

### 4.3 核心用户旅程

1. **导出**：用户点击「文件」→ Dialog 打开 → 点击「导出」→ 系统执行 `export`（路径与排除策略见 §9）→ 写入 IDB → 列表刷新出现新行（路径 + 时间）。
2. **导入**：用户在列表行 hover → 右侧出现「导入」→ 点击 → 系统将快照 **mount** 回容器 → **重启项目**（例如重新触发与「一键 PoC」一致的或可配置的启动链路，以实现为准）→ Toast 或 Dialog 内简要反馈成功/失败。
3. **异常**：配额不足、挂载失败时，用户看到明确提示，列表不被静默破坏。

## 5. 功能需求

### 5.1 「文件」入口与 Dialog 框架

- **描述**：在终端面板工具栏中，于 **「一键 PoC」按钮右侧**增加 **「文件」**按钮；点击打开 Dialog（标题建议：**工作区文件** 或与设计稿一致）。Dialog 可关闭；关闭不中断已发起的导出/导入（若异步，状态反馈仍在全局 Toast 体现）。
- **优先级**：Must
- **验收标准**：
  - [ ] 在 **`demos/webcontainer-openclaw`** 中，工具栏顺序为：`中止` → `一键 PoC` → **`文件`**（[`TerminalPane.svelte`](../../demos/webcontainer-openclaw/src/lib/features/terminal/components/TerminalPane.svelte)）。
  - [ ] 点击「文件」必现 Dialog；Esc / 关闭控件可关。
- **边界与异常**：`crossOriginIsolated` 等环境与一键 PoC 一致时「文件」是否禁用，与 §9 对齐（默认：**不与 PoC 按钮强行同禁用**，但若 WebContainer 未就绪则导出/导入按钮 disabled 并提示）。

### 5.2 列表与导出

- **描述**：Dialog 主体为 **已导出记录列表**（来自 IndexedDB，数据结构与技术 spec 一致）；顶部或显眼位置提供 **「导出」**。
- **优先级**：Must
- **验收标准**：
  - [ ] 列表为空时有空态文案（如「暂无导出，请点击导出」）。
  - [ ] 每条记录至少展示：**导出路径**、**导出时间**。
  - [ ] 点击「导出」生成新记录并出现在列表顶部或按时间排序的合理位置。
- **边界与异常**：导出失败（配额、树过大）有错误提示；不写入半条「孤儿」记录（实现可采用单次事务或与 spec 一致的策略）。

### 5.3 行 hover 与导入 + 重启项目

- **描述**：列表行 **hover** 时，**右侧**显示 **「导入」**（轻交互：常态不占满行，hover 再显现）。点击「导入」：**挂载**该快照到容器，并 **重新启动项目**。
- **优先级**：Must
- **验收标准**：
  - [ ] 非 hover 状态下行布局不拥挤；hover 后「导入」可见且可点击。
  - [ ] 导入成功后，用户可继续终端/PoC 相关操作（例如再次运行 dev / PoC），行为与「从零挂载后启动」等价或可接受偏差（见 §9）。
  - [ ] 导入失败时列表数据不丢失，错误可感知。
- **边界与异常**：导入是否需二次确认（§9）；是否与当前未保存内存态冲突（§9）。

### 5.4 与设计体系一致

- **描述**：按钮、Dialog、列表视觉遵循 **`docs/design/BitsUI.md`**（实现阶段对照 Figma）。
- **优先级**：Should

## 6. 数据与状态

- **持久化**：IndexedDB，键空间、快照形状与 [`2026-05-01_19-00_WebContainer-工作区导出导入功能.md` §4.10](../specs/2026-05-01_19-00_WebContainer-工作区导出导入功能.md) 对齐；PRD 不重复定义 schema。
- **列表排序**：默认 **导出时间降序**（最新在上），除非 UX 另有指定。
- **展示字段**：「导出路径」为导出时捕获的根路径字符串（若技术实现为 workspace 相对路径，需在列表中与用户心智一致）。

## 7. 非功能需求

- **性能**：大仓库导出可能耗时；Dialog 内需 **loading/disabled** 防重复点击（Should）。
- **安全与隐私**：IDB 仅存本地；PRD 假设 demo 可能含敏感文件时，Dialog 或 README **Should** 有一句风险提示（不阻断 MVP）。
- **可访问性**：Dialog 焦点陷阱、列表行 hover 对应键盘焦点策略（Could，与 Bits UI 组件能力对齐）。

## 8. 里程碑与依赖

- **依赖**：IndexedDB 存储 API 实现与 WebContainer `export`/`mount` 编排（技术 spec 定稿并实现）。
- **MVP**：本 PRD §5.1～5.3 全流程可走通。

| O1 | **导出根路径**：默认整棵工作区根还是可选子路径？默认排除 `node_modules` 等？ | 配额与耗时 | 产品+研发 | 开放 |
| O2 | **「重新启动项目」精确定义**：等价于再次执行「一键 PoC」脚本链路？还是先 `mount` 再由用户手动 `npm run dev`？ | 导入后验收口径 | 产品 | 开放 |
