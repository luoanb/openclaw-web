| 属性 | 内容 |
|------|------|
| 状态 | 草稿 |

## 范围（极简）

**包含**：新建 `web-claw` 正式应用的第一阶段交互范围；以 BrowserPod 为底层运行时；优先打通运行启停、运行状态、API Key / 隔离环境提示、终端创建、终端输入输出、终端切换、终端关闭、前台任务中止。

**不包含**：完整文件树、文件编辑器、命令驱动文件管理、Portal 服务预览多 tab、workspace 快照列表、BrowserPod 以外 runtime 实现、`packages/web-os` 迁移。

## 功能与交互

### 1. 应用壳与运行入口

| 项目 | 说明 |
|------|------|
| **位置** | 未来 `apps/web-claw` 主页面；页面只做交互编排，运行能力通过 `packages/os-core` 契约与 `packages/browserpod` 实现获得。 |
| **主区域** | 第一阶段默认进入「运行」视图；页面需要能容纳终端列表、当前终端面板、运行状态提示。 |
| **启动入口** | 用户点击「启动」或等价主按钮后，系统开始初始化 BrowserPod。 |
| **停止入口** | 启动成功后展示「停止」或等价操作；点击后进入停止中状态，完成后终端不可继续输入。 |
| **状态展示** | 至少展示：未启动、启动中、运行中、停止中、已停止、启动失败。 |
| **环境前置** | 若缺少 API Key、未满足 `crossOriginIsolated`、浏览器不支持，应在运行入口附近明确提示，并禁用启动操作。 |

**验收**

- [ ] 用户能在主页面看到运行状态和启动入口。
- [ ] 缺少 API Key 或隔离环境不满足时，启动入口不可误触成功，并有可理解提示。
- [ ] 启动中、运行中、停止中、失败状态均可被用户感知。
- [ ] 停止后，终端输入不会继续写入已失效会话。

---

### 2. 单终端基础交互

| 项目 | 说明 |
|------|------|
| **创建时机** | BrowserPod 启动成功后，系统自动创建第一个终端；若自动创建失败，用户可通过明确操作重试。 |
| **终端显示** | 当前终端占据主要工作区，支持展示命令输出、错误输出和运行提示。 |
| **输入行为** | 用户在终端聚焦后可输入文本；输入应被发送到当前终端会话或当前运行命令。 |
| **聚焦行为** | 切回终端视图或切换终端后，应尽量自动聚焦当前终端。 |
| **不可用态** | 未启动、启动中、停止中、已停止、终端异常时，终端输入应禁用或给出明确反馈。 |

**验收**

- [ ] 启动成功后至少有一个可见终端。
- [ ] 用户能在当前终端输入，并看到对应输出或错误反馈。
- [ ] 终端不可用时，不会静默吞输入；用户能知道为何不可输入。
- [ ] 当前终端区域在切换回来后可继续承接键盘输入。

---

### 3. 多终端管理

| 项目 | 说明 |
|------|------|
| **入口** | 终端区域提供「新建终端」入口；仅在运行中可用。 |
| **列表** | 终端以 tab、侧栏或等价列表展示；每个终端至少有名称与状态。 |
| **切换** | 点击某个终端项后，该终端成为当前终端；非当前终端不抢占主输入焦点。 |
| **命名** | 默认名称可为「终端 1」「终端 2」；支持重命名为 Should，不阻断 MVP。 |
| **关闭** | 用户可关闭非唯一终端；关闭运行中终端时需提示风险或执行明确的终止策略。 |
| **唯一终端** | 只剩一个终端时，可禁止关闭，或允许关闭后展示空态与「新建终端」入口；二选一需在实现前拍板。 |

**验收**

- [ ] 运行中可新建多个终端，并能切换当前终端。
- [ ] 每个终端的名称与状态可见。
- [ ] 关闭终端不会影响其他终端的展示与切换。
- [ ] 关闭运行中终端时，不会造成用户误以为任务仍在安全运行。

---

### 4. 命令运行与中止

| 项目 | 说明 |
|------|------|
| **运行方式** | 第一阶段仅要求终端内可运行命令；是否提供「一键运行示例项目」按钮暂不作为 Must。 |
| **前台任务** | 当前终端存在前台任务时，应展示任务运行中状态，并避免重复触发冲突操作。 |
| **中止入口** | 提供「中止」或等价操作，用于终止当前终端的前台任务或向会话发送中断信号。 |
| **中止反馈** | 中止成功、失败或不适用时均需有终端输出或 Toast 提示。 |
| **并发边界** | 多终端可并行存在；是否允许多个终端同时跑前台任务由 `os-core` 契约与 BrowserPod 能力确认后决定。 |

**验收**

- [ ] 当前终端存在运行中任务时，用户能看出它正在运行。
- [ ] 用户能对当前终端触发中止操作。
- [ ] 中止操作结果可见，不静默失败。
- [ ] 多终端切换时，中止操作只作用于当前终端或明确提示作用范围。

---

### 5. 错误与恢复

| 项目 | 说明 |
|------|------|
| **启动失败** | 展示可读错误；常见类型包括 API Key 缺失/无效、隔离环境不足、BrowserPod boot 失败。 |
| **终端失败** | 终端创建失败、会话退出、命令失败时，应在对应终端或全局状态中反馈。 |
| **重试** | 启动失败后保留「重试」入口；终端创建失败后保留「重新创建终端」入口。 |
| **停止后恢复** | 停止后用户可重新启动；旧终端是否保留历史输出待定。 |

**验收**

- [ ] 启动失败后能重试，不需要刷新页面才能恢复。
- [ ] 终端级错误归属到具体终端或当前终端，不与全局运行状态混淆。
- [ ] 停止后重新启动的入口清晰。

---

## 数据与展示约定

- 终端展示字段：终端名称、当前状态、是否当前选中、是否有运行中任务。
- 运行状态展示字段：runtime 状态、最近错误、是否可启动、是否可停止。
- API Key 不在界面明文展示；如需提示，只展示「未配置」或「校验失败」等状态。
- 技术 schema、事件名、错误类型以 `os-core` 契约文档为准，本 Light PRD 不定义接口细节。

## 待定

| 项 | 说明 |
|----|------|
| 终端实现形态 | BrowserPod 是否支持长驻交互 shell；若不支持，第一阶段是否以 `pod.run` + SDK 默认终端作为降级。 |
| 停止语义 | BrowserPod 是否存在明确 dispose/stop；若无，产品上如何表达「停止」。 |
| 唯一终端关闭 | 禁止关闭最后一个终端，还是允许关闭后展示空态。 |
| 多终端并发 | 是否允许多个终端同时运行前台任务。 |
| 示例命令入口 | 第一阶段是否内置「运行示例项目」按钮。 |
| 历史输出保留 | 停止/重启后是否保留旧终端输出。 |
