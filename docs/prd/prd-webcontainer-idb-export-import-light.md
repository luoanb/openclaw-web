## 范围（极简）

**包含**：终端工具栏「一键 PoC」右侧 **「文件」** → Dialog：**列表（IDB）** + **导出类型 Dropdown** + **导出**；列表展示 **导出类型**；行 **hover** 露 **导入** → **mount + 重启项目**；**导入不论成败均不删除** IDB 记录。

**不包含**：云同步、JSON 路径下二进制字节级可逆、多 Tab 并发合并、自动定时导出（仅 Dialog 内显式导出）。

## 功能与交互

### 1. 「文件」入口与 Dialog

| 项目 | 说明 |
|------|------|
| **位置** | `demos/webcontainer-openclaw`，[`TerminalPane.svelte`](../../demos/webcontainer-openclaw/src/lib/features/terminal/components/TerminalPane.svelte) 工具栏：**中止** → **一键 PoC** → **文件**。 |
| **触发** | 点击「文件」打开 Dialog（标题如 **工作区文件**，以设计稿为准）。 |
| **关闭** | 关闭按钮 / Esc 可关；若在途异步导出/导入，**不依赖 Dialog 开着**，结果用 Toast（或等价全局反馈）告知。 |
| **就绪态** | WebContainer **未就绪** 时：Dialog 内 **导出 / 导入** disabled，附简短说明；「文件」入口是否仍可点开仅看空列表由实现定，但不可在无容器时误触成功文案。 |

**验收**

- [ ] 工具栏顺序与「文件」存在性符合上表。
- [ ] 点击「文件」必出 Dialog；Esc/关闭可关。

---

### 2. Dialog 内：列表 + 导出

| 项目 | 说明 |
|------|------|
| **布局** | Dialog 主体为 **已导出记录列表**（数据源：IndexedDB，形状对齐技术 spec）；列表上方或首屏显眼处：**导出类型** **Dropdown** + **「导出」** 按钮。组件形态与样式遵循 **`docs/design/BitsUI.md`** 中的 **Dropdown**（及 opened 状态节点）。 |
| **导出类型选择** | 用户点击「导出」前，须通过 **Dropdown** 选定 **导出类型**（可选值集合与实现及技术 spec 对齐；至少覆盖当前 PoC 可用的一种类型，如与 `export(..., { format: 'json' })` 对应的树快照）。支持 **默认选中第一项** 或 **必选一项**，以实现与设计一致为准，但须在 UI 上可见当前选项。 |
| **列表列** | 每行至少：**导出路径**、**导出时间**（本地可读时间）、**导出类型**（与导出时 Dropdown 所选一致，持久化展示）。 |
| **排序** | 默认 **按导出时间降序**（最新在上）。 |
| **空态** | 无记录时展示引导文案，例如「暂无导出，请用 Dropdown 选择导出类型后点击导出」。 |
| **导出点击** | 点击「导出」→ 按当前 Dropdown 所选 **导出类型** 执行对应导出逻辑（如调用不同 `format` 或后续扩展路径）并写入 IDB → **列表刷新**，新记录包含 **类型** 字段且出现在排序正确位置。 |
| **进行中** | 导出耗时场景：**导出** 按钮 loading 或 disabled，防重复提交（Should）。 |

**验收**

- [ ] 空态、列表列 **路径 / 时间 / 导出类型** 齐全；导出后出现新行且顺序正确。
- [ ] 导出前存在 **导出类型** **Dropdown**，且新写入记录的列表 **导出类型** 与提交时 Dropdown 所选一致。
- [ ] 导出失败（配额、过大等）有明确错误提示，**不出现半截无效列表项**。

---

### 3. 列表行：Hover 与导入（含重启）

| 项目 | 说明 |
|------|------|
| **常态** | 行内主要展示路径与时间；**不默认铺满「导入」**，避免拥挤。 |
| **Hover** | 鼠标悬停行时，**右侧** 轻量露出 **「导入」**（文字按钮或弱化按钮）。 |
| **点击导入** | **将该条快照 mount 回容器**，并 **重新启动项目**（与「一键 PoC」同源链路还是仅 mount + 提示用户手动跑命令：**待定**，见文末）。 |
| **结果** | 成功/失败均需可感知（Toast 或 Dialog 内简短状态）。**导入不论成功或失败，均不得删除、覆盖或隐式移除该条 IDB 记录**（仅允许用户另行提供「删除快照」类显式操作，本期未要求则禁止任何导入副作用删记录）。 |
| **二次确认** | 是否覆盖当前挂载：**待定**（文末）。 |

**验收**

- [ ] 非 hover 与 hover 的布局符合「轻交互」描述；hover 下「导入」可点。
- [ ] 导入成功后可继续终端侧操作；失败有提示且列表仍在。
- [ ] **导入成功或失败后**，对应行的 IDB 记录仍存在且列表仍展示该行（**不得因导入流程删除记录**）。

## 数据与展示约定（仅交互相关）

- 「导出路径」字符串与导出时所选根一致；若为相对路径，需与用户心智一致（实现与技术 spec 对齐）。
- **导出类型**：每条快照须持久化可选枚举值（展示名与存储值以实现为准），并与 Dropdown 选项一一对应；新增类型时列表与 Dropdown 同步扩展。
- 持久化 schema **不以本 Light 文档为准**，以功能技术 spec **§4.10** 为准（字段变更须 **先修订该 spec 再落库**，避免 PRD 与契约冲突）。
